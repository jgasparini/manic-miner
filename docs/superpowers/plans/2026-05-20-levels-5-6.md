# Levels 5 & 6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create `level-05.json` and `level-06.json` so the six-level game is complete end-to-end.

**Architecture:** Each level is a Tiled-format JSON tilemap (25×15 tiles, 32 px each). The `LevelManager` loads them by key from `constants.js`'s `LEVELS` array, which already lists `'level-05'` and `'level-06'`. No engine changes needed — just the two data files.

**Tech Stack:** Tiled JSON format, Phaser 3, Vite dev server (`npm run dev`)

---

## Reference

### Coordinate system
- Grid: 25 cols × 15 rows, tile size 32 px
- Pixel position: `x = col × 32`, `y = row × 32`
- Object `y` = **top** of the 32×32 bounding box (LevelManager adds `height/2` internally)
- Items and enemies live in the **air row above** their landing platform

### Layer names required by LevelManager
`Decoration` (tilelayer), `Ground` (tilelayer), `Collectibles` (objectgroup), `Enemies` (objectgroup), `Portal` (objectgroup), `PlayerStart` (objectgroup)

### Tileset reference
- File: `../tilesets/tileset-cave.png` (relative to tilemaps dir)
- 8 tiles wide, tile ID `1` = first tile (solid ground)

### Enemy custom property
```json
{ "name": "patrolDistance", "type": "int", "value": N }
```
